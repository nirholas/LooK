# Support

How to get help with LooK and report issues.

## Getting Support

### Documentation

Start by searching this documentation using the search bar at the top.

### GitHub Issues

For bugs, feature requests, and questions:

[:octicons-mark-github-16: Open an Issue](https://github.com/nirholas/LooK/issues/new){ .md-button }

### Before Opening an Issue

1. **Search existing issues** - Your question may already be answered
2. **Check the FAQ** - Common questions are covered there
3. **Try troubleshooting** - Review the troubleshooting guide

### What to Include

When opening an issue, include:

```markdown
**Environment:**
- LooK version: (run `look --version`)
- Node.js version: (run `node --version`)
- OS: (e.g., macOS 14.0, Ubuntu 22.04, Windows 11)
- Browser: (for web editor issues)

**Description:**
A clear description of the issue.

**Steps to Reproduce:**
1. Run `look demo ...`
2. See error

**Expected Behavior:**
What should happen.

**Actual Behavior:**
What actually happens.

**Error Message:**
```
Paste the full error message here
```

**Additional Context:**
Any other relevant information.
```

## Bug Reports

For bugs:

1. Provide a minimal reproduction case
2. Include the full error stack trace
3. Note if the issue is consistent or intermittent

## Feature Requests

For new features:

1. Describe the use case
2. Explain why existing features don't solve the problem
3. Provide examples if possible

## Security Issues

For security vulnerabilities:

**Do not open a public issue.**

Email security concerns to the maintainers directly or use GitHub's private vulnerability reporting.

See [SECURITY.md](https://github.com/nirholas/LooK/blob/main/SECURITY.md) for details.

## Contributing

Want to contribute a fix or feature?

See the [Contributing Guide](../contributing.md) for how to get started.

## Commercial Support

LooK is open source and community-supported. For commercial support inquiries, open an issue with the `commercial` label.
